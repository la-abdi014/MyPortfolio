Synopsis

This project is a task manager application that allows the user to record and keep track of daily tasks at hand, creating organization and visual checklists to fulfill to-do list

Project Purpose and Goal

The purpose of this project is to ensure tasks are completed in a timely fashion, the ability to make room for other tasks, and generating a sense of responsibility prioritizing more important tasks first. The project contains features such as the task description, status, who the task is assigned to, and the ability to delete a task once finished. The project also consists of statuses from to-do, in progress, done, or on hold.

Technology Used and Why?

The technology used for this project was React and JavaScript for the front end and CRUD operations for creating, reading, updating, and deleting tasks. CSS3 for styling the webpage.

Problem Solving Technique & Optimization

The problem solving technique used for this project was to understand the users point of view and creating an MVP that delegates tasks to the user to complete in a timely fashion. The optimization that I would apply for this project would be to allow the user to have a daily view of all tasks as well as tasks completed in a week, month, and year. I would also have a set reminder, where the user is prompted everyday that a task has not been completed within a users inputted given timeframe, ex: user inputs a deadline of 2 weeks for a specific task.
